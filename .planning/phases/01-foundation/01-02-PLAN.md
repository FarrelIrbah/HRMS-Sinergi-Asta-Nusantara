  <name>Task 1: Configure NextAuth v5 with credentials provider and type augmentation</name>
  <files>
    src/lib/auth.config.ts
    src/lib/auth.ts
    src/app/api/auth/[...nextauth]/route.ts
    src/types/next-auth.d.ts
    src/middleware.ts

  <action>
    1. Create `src/types/next-auth.d.ts` to augment NextAuth types. Add `id` (string) and `role` (Role from @prisma/client) to both the `User` interface and the `Session.user` interface. Also augment the JWT type to include `id` and `role`. This is CRITICAL -- without it, role is not available in the session.

    2. Create `src/lib/auth.config.ts` -- this is the Edge-compatible config (no Prisma, no bcrypt). Export `authConfig` with:
    - pages: { signIn: "/login" }
    - callbacks.authorized: Check if user has a token. If accessing /login and already authenticated, redirect to /dashboard. If accessing protected routes without token, redirect to /login. Public paths: /login, /api/auth.
    - providers: [] (empty here, added in auth.ts)

    3. Create `src/lib/auth.ts` -- full NextAuth config. Import authConfig and extend it:
    - Add CredentialsProvider: accepts email + password, looks up user via `prisma.user.findUnique({ where: { email } })`, verifies password with `bcryptjs.compare()`, checks `isActive === true` (reject deactivated users), returns user object with id, name, email, role.
    - JWT callback: On initial sign-in (when `user` is present), copy `user.id` and `user.role` into the token. On subsequent calls, pass through.
    - Session callback: Copy `token.id` as `session.user.id` and `token.role` as `session.user.role`.
    - Export: `auth`, `signIn`, `signOut`, `handlers` from `NextAuth(config)`.
    - Use `@auth/prisma-adapter` as adapter BUT with `session: { strategy: "jwt" }` (credentials provider requires JWT strategy, not database sessions).

    IMPORTANT: Import from "next-auth" NOT "next-auth/next". Use `auth()` NOT `getServerSession()`. Auth.js v5 syntax.

    4. Create `src/app/api/auth/[...nextauth]/route.ts`:
    ```typescript
    import { handlers } from "@/lib/auth"
    export const { GET, POST } = handlers
    ```

    5. Create `src/middleware.ts` at the project root (NOT inside src/app):
    ```typescript
    import NextAuth from "next-auth"
    import { authConfig } from "@/lib/auth.config"

    export default NextAuth(authConfig).auth

    export const config = {
      matcher: ["/((?!api/auth|_next/static|_next/image|favicon.ico).*)"],
    }
    ```
    This protects all routes except auth API and static assets.
  </action>
  <verify>
    - `npx tsc --noEmit` passes with no type errors in auth files
    - `npm run dev` starts without errors
    - Navigating to `localhost:3000/dashboard` redirects to `/login`
    - The NextAuth API route responds: `curl http://localhost:3000/api/auth/providers` returns JSON with credentials provider
  </verify>
  <done>
    NextAuth v5 configured with credentials provider. JWT strategy with role in session. Middleware protects all dashboard routes. Type augmentation exposes role and id in session.
  </done>
