  <action>
    1. Initialize Prisma: `npx prisma init` (if not already done by dependency install).

    2. Create `prisma/schema.prisma` with the COMPLETE Phase 1 schema from the research document. Include ALL models:
    - User (id cuid, name, email unique, hashedPassword, role enum, isActive boolean, timestamps)
    - Department (id, name, description optional, deletedAt for soft delete, timestamps, positions relation)
    - Position (id, name, departmentId FK to Department, deletedAt, timestamps)
    - OfficeLocation (id, name, address optional, allowedIPs String[], latitude/longitude Float optional, radiusMeters Int optional, deletedAt, timestamps)
    - LeaveType (id, name, annualQuota Int, isPaid Boolean default true, genderRestriction String optional, deletedAt, timestamps)
    - AuditLog (id, userId FK to User, action enum CREATE/UPDATE/DELETE, module String, targetId String, oldValue Json optional, newValue Json optional, createdAt, indexes on userId+module+createdAt)
    - Role enum: SUPER_ADMIN, HR_ADMIN, MANAGER, EMPLOYEE
    - AuditAction enum: CREATE, UPDATE, DELETE
    Use @@map for table names (snake_case: users, departments, positions, office_locations, leave_types, audit_logs).

    3. Run `npx prisma migrate dev --name init` to create the initial migration. Ensure PostgreSQL is running locally.

    4. Create `src/lib/prisma.ts` -- Prisma client singleton using globalThis pattern to prevent hot-reload connection exhaustion:
    ```typescript
    import { PrismaClient } from '@prisma/client'

    const globalForPrisma = globalThis as unknown as { prisma: PrismaClient }

    export const prisma = globalForPrisma.prisma || new PrismaClient()

    if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma
    ```

    5. Create `src/types/index.ts` with shared types:
    - Re-export Role and AuditAction from @prisma/client
    - Define a UserSession type: { id: string, name: string, email: string, role: Role }
    - Define a PaginatedResponse<T> type: { data: T[], total: number, page: number, pageSize: number }
    - Define a ServiceResult<T> type: { success: boolean, data?: T, error?: string }

    6. Create `src/lib/constants.ts` with:
    - ROLES object mapping Role enum to Indonesian labels: { SUPER_ADMIN: "Super Admin", HR_ADMIN: "Admin HR", MANAGER: "Manajer", EMPLOYEE: "Karyawan" }
    - MODULES object for audit log module names: { USER: "Manajemen Pengguna", DEPARTMENT: "Departemen", POSITION: "Jabatan", OFFICE_LOCATION: "Lokasi Kantor", LEAVE_TYPE: "Jenis Cuti", AUTH: "Autentikasi" }
    - AUDIT_ACTIONS mapping: { CREATE: "Buat", UPDATE: "Ubah", DELETE: "Hapus" }
    - DEFAULT_PAGE_SIZE = 25
    - PAGE_SIZE_OPTIONS = [25, 50]

    7. Create `prisma/seed.ts` with a basic seed that creates ONE Super Admin user:
    - Email: admin@ptsan.co.id
    - Password: hashed "Admin123!" using bcryptjs
    - Name: "Super Admin"
    - Role: SUPER_ADMIN
    Use `upsert` to be idempotent. Add the seed script to package.json: `"prisma": { "seed": "npx tsx prisma/seed.ts" }`. Install tsx as dev dependency: `npm install -D tsx`.

    8. Run `npx prisma db seed` to seed the database.
  </action>
