  <action>
    1. Create `src/lib/validations/master-data.ts`:
    - `departmentSchema`: name (string, min 2, max 100, "Nama departemen minimal 2 karakter"), description (string, optional, max 500)
    - `positionSchema`: name (string, min 2, max 100, "Nama jabatan minimal 2 karakter"), departmentId (string, min 1, "Departemen wajib dipilih")
    - `officeLocationSchema` and `leaveTypeSchema` -- define these NOW but they will be used in Plan 07:
      - `officeLocationSchema`: name (string, min 2), address (string, optional), allowedIPs (array of strings, optional), latitude (number, optional), longitude (number, optional), radiusMeters (number, optional, min 50, max 10000)
      - `leaveTypeSchema`: name (string, min 2), annualQuota (number, min 0, max 365), isPaid (boolean), genderRestriction (string, optional, enum "MALE" | "FEMALE" or null)
    - Export all types: DepartmentInput, PositionInput, OfficeLocationInput, LeaveTypeInput

    2. Create `src/lib/services/master-data.service.ts`:
    Department functions:
    - `getDepartments(params)`: Paginated list with search by name. ALWAYS filter `where: { deletedAt: null }`. Include position count. Returns PaginatedResponse.
    - `getAllDepartments()`: Non-paginated list for dropdowns. Filter `deletedAt: null`. Returns `{ id, name }[]`.
    - `createDepartment(data, actorId)`: Create department, call createAuditLog (module: MODULES.DEPARTMENT, action: CREATE).
    - `updateDepartment(id, data, actorId)`: Fetch old values first, update, call createAuditLog with oldValue/newValue.
    - `deleteDepartment(id, actorId)`: SOFT DELETE -- set `deletedAt: new Date()`. Check if department has active positions first -- if yes, reject with "Departemen memiliki jabatan aktif, tidak dapat dihapus". Call createAuditLog (action: DELETE).

    Position functions:
    - `getPositions(params)`: Paginated list with search by name. Filter `deletedAt: null`. Include department relation (name). Returns PaginatedResponse.
    - `getAllPositions(departmentId?)`: Non-paginated list for dropdowns. Filter `deletedAt: null`. Optional filter by departmentId. Returns `{ id, name, departmentId }[]`.
    - `createPosition(data, actorId)`: Verify department exists and is not deleted. Create position, call createAuditLog.
    - `updatePosition(id, data, actorId)`: Fetch old, update, audit log.
    - `deletePosition(id, actorId)`: SOFT DELETE. Call createAuditLog. (In Phase 2, check for active employees before allowing delete.)

    CRITICAL: Every query on Department and Position must include `where: { deletedAt: null }` unless specifically looking up a deleted record.

    3. Create `src/lib/actions/master-data.actions.ts` -- Server Actions:
    - All actions: check session, verify SUPER_ADMIN role, validate with Zod, call service, revalidate "/master-data".
    - Department actions: `createDepartmentAction`, `updateDepartmentAction`, `deleteDepartmentAction`
    - Position actions: `createPositionAction`, `updatePositionAction`, `deletePositionAction`
    - Leave space for office location and leave type actions (Plan 07 will add them to this file).
    - Return `{ success: true }` or `{ success: false, error: "message" }`.
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - All service functions properly filter by `deletedAt: null`
    - Department delete checks for active positions
    - All mutations call createAuditLog
  </verify>
  <done>
    Validation schemas, service layer, and server actions for departments and positions. Soft delete implemented. Audit logging on all mutations. Department-position relationship enforced.
  </done>
