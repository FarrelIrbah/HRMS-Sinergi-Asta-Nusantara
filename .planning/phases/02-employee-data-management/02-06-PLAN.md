     - Get their Employee record via getEmployeeByUserId(session.user.id)
     - If employee not found OR employee.id !== params.id, redirect to /dashboard (cannot view others)
     - Mode: "readonly"

     - Check canManagerAccessEmployee(session.user.id, params.id)
     - If false, redirect to /employees (cannot view employees outside their department)
     - Mode: "readonly"

     **HR_ADMIN / SUPER_ADMIN:**
     - Mode: "edit"

   - Fetch full employee data via getEmployeeById(params.id)
   - If employee not found, return notFound()
   - Fetch departments and positions lists (for edit mode dropdowns)
   - Render page header showing employee name + NIK, with back link to /employees

All three tabs follow the same pattern:
- "use client" component
- Accept props: { employee, readOnly: boolean } (plus departments/positions for employment tab)
- Use react-hook-form with the appropriate Zod resolver
- Pre-populate form with employee data (defaultValues from employee prop)
- If readOnly: all inputs are disabled, save button is hidden
- If editable: save button calls the appropriate server action
- On success: toast("Data berhasil disimpan"), router.refresh() to re-fetch data
- On error: toast.error(result.error)

- Fields: namaLengkap (required), nikKtp (optional, maxLength 16 with validation), tempatLahir, tanggalLahir (date input), jenisKelamin (select from Gender), statusPernikahan (select from MaritalStatus), agama (select from Religion), alamat (textarea), nomorHp, email (readonly -- cannot change email after creation)
- Uses updatePersonalInfoSchema
- Calls updatePersonalInfoAction(employee.id, formData)
- Use GENDER_LABELS, RELIGION_LABELS, MARITAL_STATUS_LABELS from constants for select options

**EmploymentDetailsTab** (`employment-details-tab.tsx`):
