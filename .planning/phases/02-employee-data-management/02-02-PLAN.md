  <action>
Create `src/lib/services/employee.service.ts` following the exact pattern of master-data.service.ts and user.service.ts.

Functions to implement:

1. `generateEmployeeNIK()` -- Generate format EMP-{YYYY}-{4-digit sequential}. Find the last employee with matching year prefix, increment. Handle inside transaction to prevent race conditions.

2. `getEmployees(params)` -- Paginated list with filters:
   - params: { page, pageSize, search, departmentId?, positionId?, isActive?, contractType? }
   - search: filter by namaLengkap OR nik (case-insensitive contains)
   - Include department and position names in results (select: { id, name })
   - Return { data, total, page, pageSize, totalPages }

3. `getEmployeesForManager(userId)` -- Look up manager's Employee record to find their departmentId, then call getEmployees scoped to that department. Return empty result if manager has no Employee record.

4. `getEmployeeById(id)` -- Full employee with department, position, documents, and emergencyContacts included. Return null if not found.

5. `getEmployeeByUserId(userId)` -- Find employee by userId. Used for Employee self-view and Manager department lookup.

6. `createEmployee(data, actorId)` -- CRITICAL: Use prisma.$transaction():
   a. Check email uniqueness (User table)
   b. Generate NIK inside transaction
   c. Hash initialPassword with bcryptjs (import from existing pattern in user.service.ts)
   d. Create User with role EMPLOYEE, isActive true
   e. Create Employee linked to User
   f. Outside transaction: createAuditLog with action CREATE, module MODULES.EMPLOYEE
   g. Return ServiceResult<Employee>

7. `updatePersonalInfo(employeeId, data, actorId)` -- Update personal fields, audit log with old/new values.

8. `updateEmploymentDetails(employeeId, data, actorId)` -- Update employment fields, audit log.

9. `updateTaxBpjs(employeeId, data, actorId)` -- Update tax/BPJS fields, audit log.

10. `deactivateEmployee(employeeId, data, actorId)` -- CRITICAL: Use prisma.$transaction():
    a. Update Employee: isActive false, terminationDate, terminationReason
    b. Update linked User: isActive false
    c. Audit log with action UPDATE

11. `canManagerAccessEmployee(managerUserId, employeeId)` -- Check if the employee belongs to the same department as the manager. Return boolean.

Follow the existing ServiceResult<T> return type pattern from types/index.ts.
Use createAuditLog from @/lib/prisma for all mutations.
Import MODULES from constants.

async function requireHRAdmin(): Promise<ServiceResult<{ userId: string }>> {
  const session = await auth();
  if (!session?.user) return { success: false, error: "Sesi tidak valid" };
  if (session.user.role !== "HR_ADMIN" && session.user.role !== "SUPER_ADMIN") {
    return { success: false, error: "Akses ditolak" };
  }
  return { success: true, data: { userId: session.user.id } };
}
